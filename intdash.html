📐 Scoring Methodology & Workflow

How the Matchmaker ranks candidates and how requests flow through the system.

Match Score Formula (0–100)
Match Score = Proficiency (0–35) + Availability (0–35) + Cross-Skill (0–20) + Proximity (0–10)

Proficiency = (skill_level / max_level) × 35   [Expert=35, Proficient=26, Developing=17]
Availability = ((100 - weighted_util) / 100) × 35   [capped at 0 if overloaded]
Cross-Skill = secondary_process_count × relevance_weight × 20
Proximity = same_manager: 10 · same_org: 7 · cross-org: 4

SLA Thresholds & Automation
🚨
SLA Breach Escalation
If request sits past SLA target → auto-escalate to Approver's manager with full context.
IF time_elapsed > SLA_target THEN escalate(approver.manager)
⚠️
Capacity Conflict Detection
If deployment would push resource above 95% weighted util → flag for review before deployment.
IF resource.util + hours > 95% THEN flag("Conflict")
📦
Auto-Close
Deployed requests auto-close 3 business days after end date if no further action.
IF status="Deployed" AND date > end + 3d THEN close()
📊
Weekly Digest
Summary email every Monday — open requests, avg response time, SLA compliance, bottlenecks.
CRON: Monday 9AM → email(approvers, summary)
🔔
Deployment Confirmation Nudge
If POC hasn't confirmed within 24 hrs of deployment → auto-nudge.
IF !poc_confirmed AND time > 24h THEN nudge(requestor)
🔄
Demand Spike Auto-Alert
If forecast shows capacity breach within 2 weeks → auto-generate draft request.
IF forecast[+2wk] > capacity THEN draft_request()
